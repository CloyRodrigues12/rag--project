To meet the  requirement of robustly handling structured and unstructured data, we need to configure Supabase
to store both the text we scrape from the websites and the mathematical vectors (embeddings)
that the AI will use to search that text.

Head over to your Supabase project dashboard, click on the SQL Editor in the left sidebar, and paste the following code.

-- 1. Enable the pgvector extension to handle unstructured data (embeddings)
CREATE EXTENSION IF NOT EXISTS vector;

-- 2. Create a table to store structured metadata (URL) and unstructured text/vectors
CREATE TABLE documents (
  id BIGSERIAL PRIMARY KEY,
  url TEXT NOT NULL,
  content TEXT NOT NULL,
  embedding VECTOR(1536) -- 1536 dimensions is the standard for most LLM embeddings like OpenAI's
);

-- 3. Create a similarity search function (This handles the "Retrieval" in RAG)
CREATE OR REPLACE FUNCTION match_documents (
  query_embedding VECTOR(1536),
  match_threshold FLOAT,
  match_count INT
)
RETURNS TABLE (
  id BIGINT,
  url TEXT,
  content TEXT,
  similarity FLOAT
)
LANGUAGE SQL STABLE
AS $$
  SELECT
    documents.id,
    documents.url,
    documents.content,
    1 - (documents.embedding <=> query_embedding) AS similarity
  FROM documents
  WHERE 1 - (documents.embedding <=> query_embedding) > match_threshold
  ORDER BY similarity DESC
  LIMIT match_count;
$$;


Database Adjustment
when we set up the Supabase database earlier, we formatted the vector column to hold 1536 dimensions (which is the size OpenAI uses).
 Our open-source model produces 384 dimensions.

Go back to your Supabase SQL Editor and run this quick update to fix the dimensions:

-- Drop the old function
DROP FUNCTION IF EXISTS match_documents;

-- Change the embedding column to 384 dimensions
ALTER TABLE documents ALTER COLUMN embedding TYPE vector(384);

-- Recreate the search function with 384 dimensions
CREATE OR REPLACE FUNCTION match_documents (
  query_embedding VECTOR(384),
  match_threshold FLOAT,
  match_count INT
)
RETURNS TABLE (id BIGINT, url TEXT, content TEXT, similarity FLOAT)
LANGUAGE SQL STABLE AS $$
  SELECT id, url, content, 1 - (embedding <=> query_embedding) AS similarity
  FROM documents
  WHERE 1 - (embedding <=> query_embedding) > match_threshold
  ORDER BY similarity DESC
  LIMIT match_count;
$$;
